현재의 PPAM 프로젝트에 대해 아래 3개에 대해 확인하자. 
1. 현재 프로젝트 진행된 것
2. 지난번 @docs\ppam_design_review_summary.md 의 디자인리뷰 후 피드백
3. 이번 @docs\251025_vc_teaching_organized.md 의 스타트업 CTO의 피드백
이렇게 3개의 것을 고려하여 현재의 PPAM이 첫번째 마일스톤까지 진행해야할 것 결정해야할 컨셉에 대해 고민하려고 한다. 이미 충분히 좋은 아이디어는 나왔고 대신에 마일스톤 1단계를 하기에 모드 아이디어를 적용할 수 없고 오히려 지금 적용한 것들 중에서도 빼야할 수도 있기에 그런것들을 고려해야한다. 

너가 진행된것, 더할것, 뺼것등을 나에게 여러가지 질문을 통해 판단하여 최종적으로 어떻게 하는것이 좋은지 3가지 안정도로 정리될수 있게 해보자. 

그리고 현재 flutter + dart 기반으로 되어있는데 이를 react로 짜는것은 어떤지도 고려해보자. 범용성이나 추후 개발자를 구하는데 flutter는 사용자가 별로 없을 것 같은데 react는 많기에 그런 측면에서 유리할 것 같아서 그렇다. 

이내용에 대해서도 오늘날짜로 docs 폴더에 마크다운으로 남기자 .
